## 摘要
大规模视频检索对于自动驾驶中的数据筛选和安全验证至关重要，用户不仅需要查找场景，还要检索切入和急刹车等动态事件。现有的视觉-语言和基于关键词的检索方法常遗漏这些事件，因为相关运动未在文本中明确描述或无法通过词汇重叠捕获。基于规则的检索能更直接编码此类事件，但方法脆弱：当生成或手写规则的假设与真实驾驶数据不符时，规则往往失效。我们提出STRIVE-D，一个针对驾驶视频的数据校准检索框架。它利用弱标注域内视频评估查询规则的可靠性，调整与观测数据不匹配的规则，并将校准后的规则分数与视觉-语言和关键词检索信号融合。在三个驾驶基准（包括DrivingDojo上新发布的人工标注事件数据）上，STRIVE-D相比现有最佳方法，在top-1准确率上实现了高达84%的相对提升。

## Abstract
Video retrieval at scale is central to data curation and safety validation in autonomous driving, where users want to find not only scenes but also dynamic events such as cut-ins and hard braking. Existing vision-language and keyword-based retrieval methods often miss these events because the relevant motion may not be explicitly described in text or captured by lexical overlap. Rule-based retrieval can encode such events more directly, but it is brittle: generated or hand-written rules often fail when their assumptions do not match real driving data. We propose STRIVE-D, a data-calibrated retrieval framework for driving videos. It uses weakly labeled in-domain videos to estimate when a query rule is reliable, adapt rules that mismatch observed data, and fuse calibrated rule scores with vision-language and keyword-based retrieval signals. Across three driving benchmarks, including newly released human-annotated event data on DrivingDojo, STRIVE-D delivers up to 84% relative improvement in top-1 accuracy over state-of-the-art methods.

- **现有方法局限**：
  - **密集视觉‑语言嵌入检索**：将细粒度运动模式（急刹车、切入）混入全局场景表示，导致运动信号被“稀释”（**F1 稀释**）。
  - **基于规则的结构化接地**：可直接在逐帧感知输出上执行规则，评分视频与查询的匹配度。但大语言模型（ LLM）在生成规则时缺乏对真实感知信号分布的先验，提出的数值阈值（阈值、幅度、窗口长度）结构上合理但尺度失准（**F2 误校准**）。

- **整体含义**：需要一种既能结构化接地、又能根据真实数据自动校准，并融合多模态检索信号的方法。

## 2. 方法论：STRIVE-D 架构
### 2.1 总体框架
STRIVE-D 包含三个核心组件，分别应对三种失效模式：
- **符号路径**（应对 F1）：将查询翻译成可执行的结构化规则，对逐帧感知表示进行评分，而非依赖单一嵌入。
- **校准规则库**（应对 F2）：在辅助域内视频上，通过弱监督的排序学习（Learning‑to‑Rank, L2R）联合调整规则的数值约束，形成可复用的规则库。
- **置信度感知适配策略**（应对 F3）：在检索时，若查询匹配库内事件则直接重用校准规则；否则以最接近的校准规则为上下文示例合成新规则，保留校准尺度。
- **多源融合**：将符号分数与密集嵌入、稀疏关键词检索信号经查询条件路由融合，得到最终排序。

### 2.2 符号路径：规则设计
- **逐帧结构化表示**：每个视频被表示为逐帧记录，包含检测到的物体类别、距离、车道、横向偏移等。
- **规则组成**：
  - **实体选择器**：指定“谁”（如`class_filter: ["car","truck"]`，聚合模式`any`或`consensus`）。
  - **硬门**：布尔条件（如`distance < 40.0 m`），任意不满足即得0分。
  - **软分数**：连续打分，计算字段在指定窗口内的最大变化与期望幅度的比率（归一化到[0,1]），带方向与权重。
- **评分逻辑**：硬门过滤准入，软分数提供分级排序强度，实体选择器聚合并计算视频级分数。

### 2.3 校准库构建与适配
- **事件挖掘**：在辅助视频集上用面向运动学的视频字幕器（Qwen2.5‑VL‑7B）生成描述， LLM 提取候选运动事件（21类），构成正/负例弱监督集。
- **L2R校准**：固定规则结构，用 LLM 作为优化器迭代搜索数值约束，以平均精度（AP）为目标。每轮 LLM 提出候选参数→ AP 评分→ 反馈最优历史→ 修改，直到 AP 达标或达到最大迭代次数。
- **适配策略**： LLM 匹配最接近的库事件并输出置信度；高置信度直接重用，低置信度则以库规则为样例合成新规则。

### 2.4 多源融合
- **三种信号**：密集嵌入（Qwen3‑VL‑Embedding‑8B）、稀疏关键词（BM25 对辅助字幕）、符号规则分。
- **融合**：查询条件加权倒数排名融合（RRF），权重由“融合路由器” LLM 根据查询内容决定哪个后端更具区分力。

- **核心结论**：LLM 能可靠地将查询翻译为规则结构，但数值阈值必须通过数据校准才能有效接地。STRIVE‑D 通过 L2R 弱监督校准解决了 miscalibration，显著提升了动态事件检索的 top‑1 准确率（在 DrivingDojo 上相对嵌入最强基线提升 83.6%）。
