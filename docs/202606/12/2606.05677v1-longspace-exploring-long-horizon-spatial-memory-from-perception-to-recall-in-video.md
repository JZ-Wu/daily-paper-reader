## 摘要
多模态大语言模型（MLLMs）在图像和视频理解方面取得了进展，并越来越能够处理更长的视觉输入。自动驾驶和机器人导航等长时域任务不仅需要识别当前视图，模型还必须记忆和检索之前观察到的空间布局、路径、视角变化和物体状态。为了评估这一能力，我们引入了LongSpace-Bench，一个用于长时域空间记忆的房间导览视频基准，涵盖场景感知、空间关系和空间记忆。在本工作中，我们进一步提出了LongSpace，一个用于长视频空间推理的记忆框架。LongSpace将长视频建模为顺序块，将三维结构线索融入早期解码器层，并构建层感知记忆以实现问题引导的检索。在多个空间推理基准上的实验表明，LongSpace提升了长视频空间理解，进一步证明显式空间记忆是长时域视频MLLMs的关键能力。

## Abstract
Multimodal Large Language Models (MLLMs) have advanced image and video understanding and can increasingly handle longer visual inputs. Long-horizon tasks such as autonomous driving and robotic navigation require more than recognizing the current view, as models must remember and retrieve previously observed spatial layouts, routes, viewpoint changes, and object states. To evaluate this capability, we introduce LongSpace-Bench, a room-tour video benchmark for long-horizon spatial memory, covering scene perception, spatial relations, and spatial memory. In this work, we further propose LongSpace, a memory framework for long-video spatial reasoning. LongSpace models long videos as sequential chunks, incorporates 3D structural cues into early decoder layers, and constructs layer-aware memory for question-guided retrieval. Experiments on multiple spatial reasoning benchmarks show that LongSpace improves long-video spatial understanding, further demonstrating explicit spatial memory as a key capability for long-horizon video MLLMs.

- **研究背景**：多模态大语言模型（MLLMs）正从静态图像理解向长视频理解延伸。然而，自动驾驶、机器人导航等长时域任务不仅需要识别当前画面，还必须在整个观察序列中**记住并检索先前见过的空间布局、路线、视角变化和物体状态**。
- **核心问题**：当前模型缺乏对**长时域空间记忆**的有效支持——空间证据分散在不同时间步，简单扩展输入长度容易稀释重要线索，且非结构化的片段信息难以在后续推理中被高效检索。

## 2. 方法论

### 2.1 整体思路
LongSpace 框架围绕两个核心设计：
1. **空间结构感知（Spatial Structure Perception, SSP）**：在解码器早期层注入3D几何特征，强化局部空间表示。
2. **分层KV内存（Hierarchical KV Memory, HKM）**：将长视频划分为有序片段，为每个片段构建层感知的压缩记忆，推理时按问题检索跨片段的空间证据。

### 2.2 空间结构感知（SSP）
- 基座模型：Qwen3-VL-8B，2D视觉编码器提供视觉Token。
- 3D几何编码器：使用π3提取3D空间特征，经平均池化、长度对齐和归一化后得到与视觉Token数量匹配的3D空间Token **G**。
- 融合机制：在每个SSP注入层中，将当前视觉隐藏状态与3D空间Token进行融合，通过低秩投影产生缩放项γ、偏移项β和融合门，控制3D更新的强度；同时通过深度可分离卷积提取轻量结构残差，最后以残差形式写回视觉Token位置：  
  **HΩ_l ← HΩ_l + Δ_l(HΩ_l, G, q_l)** ，其中q_l为文本状态提供的查询上下文。
- 注入位置：仅在前8个解码器层注入，实验表明适度层数效果最佳。

### 2.3 分层KV内存（HKM）
HKM 是推理时的内存底层，在有限上下文中保留长视频证据：

- **视频分块编码**：将视频按时间顺序切分为多个片段（chunk），每片段固定帧数（32帧），片段间有4帧重叠。
- **内存层级角色**：
  - **感知内存（Sensory Memory）**：保留细粒度的视觉和空间证据。
  - **工作内存（Working Memory）**：绑定当前片段内的物体、局部空间关系和最近变化。
  - **长内存（Long Memory）**：将每片段蒸馏为稳定的时间锚点和场景线索，用于跨大时间间隔的证据定位。
- **内存更新（Writing）**：
  1. 对当前片段的各解码层KV状态和隐藏特征，根据层角色(ρ(l))进行**选择（Select）**：基于显著性、状态变化、覆盖率和新近度四种归一化信号加权评分。
  2. 将选中的条目与已有内存合并，并通过**压缩（Compress）** 保持在角色特定的预算B_{ρ(l)}内：保留高分或近期条目原始形式，其余按时序分组，使用评分权重进行加权池化压缩，保留位置ID和片段摘要。
- **内存检索与解码（Reading）**：
  - 最终问题编码为查询，在感知和工作内存中执行问题关联的top-k检索。
  - 在长内存中先通过片段摘要或空间原型粗定位相关片段，再读取压缩的KV条目。
  - 检索到的KV状态作为冻结的内存前缀注入解码器注意力计算，仅自回归缓存随生成更新。

## 3. 实验设计

### 3.1 评估基准与数据集
- **LongSpace-Bench**：自主构建，含445个真实室内房间导览视频，总时长约159小时，4,073个问答对。
- 任务分为三个层级10个子任务：
  - **场景感知**：对象计数、场景分类、场景一致性。
  - **空间关系**：相对距离、相对朝向。
  - **空间记忆**：出现顺序、状态变化、自我中心推理、路径规划、路径回忆。
- 答案形式：计数题为数值答案（MRA评估），其余多为多选（准确率评估）。
- 视频长度覆盖**短（<8min）、中（8-20min）、长（>20min）**，平均21.4分钟。

**对比基准（VSI-Bench）**：用于评估标准空间推理能力，包含8个子任务。

### 3.2 对比方法
- **专有模型**：GPT-5、Gemini-3-Pro。
- **开源视频MLLMs**：Qwen2.5-VL/3-VL系列、InternVL3.5、LLaVA-OneVision、LLaVA-NeXT-Video、LongVA、LongVILA等。
- **空间中心模型**：Spatial-MLLM、VG-LLM、VST、VLM-3R、SpatialLadder、Cambrian-S等。

输入协议按各模型最佳实践设置（如GPT-5 50帧均匀采样，Gemini 1fps，Qwen 512帧，空间模型多为32帧）。LongSpace使用1fps分块内存推理，32帧/块，4帧重叠，内存一次构建可复用回答多个问题。

## 4. 资源与算力

- 训练使用**8块NVIDIA A100 80G GPU**。
- 全局batch size 64，训练**1个epoch**。
- 优化器：AdamW，学习率1×10⁻⁵，余弦调度，warmup比率0.03。
- 视频处理：每个视频最多取32帧。

## 6. 主要结论与发现

- LongSpace-9B在LongSpace-Bench上以**49.2总分**超过最强开源基线Qwen3-VL-32B（46.5）和最强专有基线Gemini-3-Pro（45.3），尤其在**出现顺序、状态变化、路径回忆**等记忆密集型任务上优势明显。
- 相比于扩展输入或更强视觉编码器，**显式空间记忆**对长视频空间推理的提升更为关键。
- 分层内存的**层感知组织**和**角色设计**比单纯增大内存容量更重要；移除长角色会导致性能大幅下降（降至34.8）。
- HKM在视频时长增加时收益更大（长视频增益15.1点），证明其跨长时域证据保留的有效性。
- 在VSI-Bench上同样取得最优（70.8均分），表明几何感知模块对局部空间估计也有普适帮助。
