## 摘要
多模态大语言模型（MLLMs）因长视觉标记序列上的自注意力计算成本为二次方而面临严重的推理瓶颈。然而，我们发现了当前架构中的一个关键低效之处：视觉注意力饱和。我们的分析表明，视觉标记在早期层中迅速建立其空间结构和模态内关系，使得更深层中的视觉到视觉自注意力在计算上变得冗余。相反，这些层中的前馈网络（FFNs）对于将视觉特征投影到不断演化的文本语义空间中仍然至关重要。利用这一见解，我们提出了Visual-Skip（V-Skip），这是一种无需训练的推理范式，它将空间交互与语义演化解耦。V-Skip不是丢弃标记，而是通过选择性地绕过饱和的视觉自注意力模块来施加逐块结构稀疏性。此外，认识到不同的下游任务需要不同的推理深度，V-Skip采用轻量级的少样本校准来动态路由任务最优的稀疏路径。大量实验表明，V-Skip有效地绕过了冗余的视觉注意力以实现逐块稀疏性，在多种MLLMs上保持了94.16%至100.31%的性能保留率。最终，我们证明，为了更有效地推理，模型不需要丢弃它们所看到的东西——它们只需要在合适的深度“看得更少”。

## Abstract
Multimodal Large Language Models (MLLMs) face a significant inference bottleneck due to the quadratic computational cost of self-attention over long visual token sequences. However, we identify a critical inefficiency in current architectures: Visual Attention Saturation. Our analysis reveals that visual tokens rapidly establish their spatial structure and intra-modal relationships in early layers, rendering visual-to-visual self-attention in deeper layers computationally redundant. Conversely, Feed-Forward Networks (FFNs) in these layers remain essential for projecting visual features into the evolving textual semantic space. Leveraging this insight, we present Visual-Skip (V-Skip), a training-free inference paradigm that decouples spatial interaction from semantic evolution. Rather than discarding tokens, V-Skip imposes block-wise structured sparsity by selectively bypassing saturated visual self-attention modules. Furthermore, recognizing that varying downstream tasks demand distinct reasoning depths, V-Skip employs a lightweight, few-shot calibration to dynamically route the task-optimal sparsity path. Extensive experiments demonstrate that V-Skip effectively bypasses redundant vision attention to achieve block-wise sparsity, maintaining a 94.16% to 100.31% performance retention across diverse MLLMs. Ultimately, we prove that to reason more effectively, models do not need to discard what they see -- they simply need to "look less" at the right depth.

- **研究背景**：多模态大语言模型（MLLMs，如 LLaVA 系列、Qwen-VL）在推理时需处理高分辨率图像对应的巨量视觉 token（如 LLaVA-NeXT 可达 2800+ tokens）。视觉序列上的自注意力计算复杂度为 O(N²)，成为推理速度与内存的主要瓶颈。
- **现有方案的问题**：现有加速方法多采用“破坏性”策略：要么直接丢弃视觉 token（如 Token Pruning、VTW），要么跳过整个 Transformer 层（如 ShortV）。前者丢失细粒度信息并增加幻觉风险，后者忽视前馈网络（FFN）在语义演化中的关键作用，导致特征错位。
- **核心发现**：研究者发现**视觉注意力饱和**现象：视觉 token 在浅层迅速完成空间结构建立与模态内交互；在深层，视觉自注意力趋于静态、变得计算冗余，但 FFN 对将视觉特征映射到不断演化的文本语义空间仍至关重要。
- **整体含义**：提出 **V‑Skip** 范式，以“看得更少”的方式实现高效推理，揭示解耦视觉自注意力与语义投影的必要性，展示在保持全量视觉上下文的同时可通过逐块跳过饱和注意力来消除二次计算冗余。

## 2. 论文提出的方法论

- **核心思想**：不在数量上删减 token，而是在结构上对注意力计算进行块级稀疏化，选择性地绕过深层饱和的视觉自注意力模块，同时完整保留 FFN 变换。通过少量样本校准动态确定不同下游任务的最优跳过层组合。
- **关键技术细节**：
  - **视觉信息增益（VIG）**：定义层 \(l\) 在任务 \(k\) 下的任务特定 VIG 为跳过该层视觉自注意力前后的输出分布 KL 散度期望（通过 20 个少样本计算）。
    \[
    G_k(l) = \mathbb{E}_{X\sim\mathcal{D}_k} [D_{KL}(P(y|X) \| P_{\neg l}(y|X))]
    \]
  - **饱和层识别**：根据 VIG 得分，从所有层中选择 VIG 最低的 Top‑N 层组成跳过集合 \(\mathcal{L}_{skip}^{(k)}\)，最小化信息损失。
  - **分块注意掩码（Decoupled Attention Masking）**：在被标识的饱和层中，修改因果注意掩码：视觉查询 token 不允许参与其他视觉 token 的注意力计算（视觉旁路），文本 token 仍可完全访问视觉与文本上下文。公式上等价于对视觉查询施加异质注意力掩码。
  - **FFN 保留**：所有 token 仍通过 FFN 层，以保证点式非线性投影和语义轨迹对齐。
  - **复杂度降低**：标准 MHSA 复杂度从 \(O((N_v+N_t)^2)\) 降至 \(O(N_t^2 + N_t N_v)\)，在 \(N_v \gg N_t\) 时接近消除二次项。
- **算法流程**：
  1. 用目标任务的少量样本（20个）计算各层 VIG。
  2. 按 VIG 升序选择 N 个层作为跳过候选。
  3. 推理时在这些层中让视觉 token 绕过自注意力计算，文本 token 保持全注意力；所有 token 均经过 FFN。

## 6. 论文的主要结论与发现

- 深层视觉自注意力确实是冗余的，V‑Skip 通过绕过这些操作实现有效的块级稀疏，同时无损保留视觉上下文和语义对齐能力。
- 在 LLaVA 系列上平均性能保留达 98.42%–100.31%（7B 模型甚至轻微超越密集基线）；在 Qwen2.5‑VL 上保持 94.16% 的高保留率，显著优于破坏性裁剪方法。
- V‑Skip 能作为即插即用的增强器与 token 裁剪方法（如 FastV）结合，进一步提升效率而提升或保持性能。
- 相较于同等 FLOPs 的 ShortV/VTW，V‑Skip 可跳过更多层仍然保持高准确度，证明了选择性绕过视觉注意力的结构优势。
- 视觉注意力饱和是跨任务存在的；但不同任务的最优稀疏路径不同，轻量少样本校准可动态路由，提供比统一掩码更好的表现。

## 7. 优点

- **创新性**：首次将视觉自注意力冗余现象明确建模为“视觉注意力饱和”，并提出基于块级结构稀疏的解耦策略。
- **非破坏性**：不丢弃 token、不冻结层，保持完整视觉信息和语义演化路径，缓解幻觉问题。

- **自适应**：量化的 VIG 度量加上少样本校准可动态匹配不同任务所需的推理深度。
- **兼容性**：可与 token 裁剪等方法正交结合，进一步提升效率。

## 8. 不足与局限

- **加速幅度相对温和**：仅绕过视觉自注意力，保留 FFN 和文本侧全注意力，FLOPs 减少和延迟降低不如激进 token 裁剪方法（如 VTW）显著；在低分辨率场景下收益可能有限。
- **任务特定校准需求**：跨数据域泛化时最优跳过层会变化，若缺乏目标域样本则需退化为通用配置，可能损失部分精度。
- **模型架构依赖性**：Qwen2.5‑VL 由于窗口注意力和位置编码的差异，视觉饱和表现与 LLaVA 不同，需仔细调优预算 N；方法在不同架构下并非完全即插即用。
- **实验环境局限**：仅在单卡 3090 上测试推理效率，未展示批量吞吐或生产级部署下的真实加速比；高并发下效果需进一步验证。
- **评估指标聚焦性能保留**：对延迟/FLOPs 的绝对减少值分析不够深入，实时场景的综合收益需要更多实证。
